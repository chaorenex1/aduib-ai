# QA Memory Service 接口总览

## 服务简介

QA Memory Service 是一个基于 RPC 的问答记忆管理系统，用于存储、检索、验证和管理 AI 问答对。服务通过向量数据库实现语义搜索，支持多级验证和 TTL 生命周期管理。

```
│ (qa_memory_     │     │  (QAMemory-     │     │ (SQLAlchemy +   │

┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐

```

## 接口分类

### 1. 检索接口
- `retrieve_qa_kb` - 语义搜索 QA 记忆库
- `qa_detail` - 获取单个 QA 记录详情

### 2. 生命周期管理接口
- `qa_upsert_candidate` - 创建/更新 QA 候选记录
- `qa_validate_and_update` - 验证并更新 QA 记录状态
- `qa_expire` - 清理过期 QA 记录

### 3. 使用统计接口
- `qa_record_hit` - 记录 QA 使用命中（展示/使用）

## 设计原则

### 1. 命名一致性
- 公共接口以 `qa_` 前缀开头（除检索接口外）
- 参数命名统一：`namespace`（项目ID）、`qa_id`（记录ID）

### 2. 响应标准化
```python
{
    "schema_version": 1,           # 接口版本
    "results": [],                 # 数据结果
    "meta": {}                     # 元数据
}
```

### 3. 错误处理
- 通过返回 `{"status": "not_found"}` 表示资源不存在
- 不抛出异常，保持接口稳定性

### 4. 异步支持
- 所有接口均为 `async def` 定义
- 支持高并发场景

## 数据模型

### 核心实体
- **QaMemoryRecord**: QA 记录实体，包含问题、答案、元数据
- **QaMemoryEvent**: QA 事件日志，记录所有操作
- **QaMemoryStatus**: 状态枚举（候选/活跃/陈旧/废弃/冻结）
- **QaMemoryLevel**: 验证级别枚举（L0-L3）

### 字段说明
- `trust_score`: 基于验证历史的信任评分 (0.0-1.0)
- `confidence`: 初始置信度 (0.0-1.0)
- `level`: 验证级别，决定优先级和权重
- `ttl_expire_at`: TTL 过期时间，支持自动降级

## 性能特征

### 时间复杂度
- 检索: O(log n) - 向量索引搜索
- 创建: O(1) - 数据库插入
- 更新: O(1) - 主键更新

### 空间复杂度
- 每条记录: ~1-10 KB（包含向量嵌入）
- 索引大小: ~1.5x 原始数据

## 安全考虑

### 访问控制
- 项目隔离：通过 `namespace` 参数实现数据隔离
- 权限验证：由调用方负责

### 数据保护
- 敏感信息：不存储个人身份信息（PII）
- 审计日志：所有操作记录到 `QaMemoryEvent`

## 限制与约束

### 容量限制
- 单个项目最大记录数：无硬限制（建议 < 1,000,000）
- 向量维度：1536（适配 OpenAI embeddings）
- TTL 范围：14-180 天

### 性能限制
- 并发连接：依赖数据库连接池
- 响应时间：检索 < 100ms，写入 < 50ms
- 批量操作：支持批量过期，默认 200 条/批次
