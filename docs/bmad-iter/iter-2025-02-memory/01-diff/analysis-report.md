## 2. 问题识别

### 2.1 架构碎片化

| 问题 | 影响 | 严重程度 |
|------|------|----------|
| 三种记忆系统独立运行 | 无法统一管理和查询 | 🔴 高 |
| GraphMemory 未集成 | 知识图谱能力未利用 | 🔴 高 |
| QA Memory 与 Agent Memory 隔离 | 问答知识无法复用 | 🟡 中 |
| 无统一的记忆生命周期管理 | 资源浪费、数据不一致 | 🟡 中 |

### 2.2 功能缺失

| 缺失功能 | 描述 |
|----------|------|
| 跨会话记忆 | Agent 记忆限于单会话 |
| 用户画像记忆 | 无持久化用户偏好 |
| 情景记忆 | 无事件时间线记忆 |
| 记忆索引策略 | 单一向量检索不足 |
| 记忆压缩/遗忘 | 无长期记忆管理 |

### 2.3 存储层问题

```
当前状态:
┌───────────────────────────────────────────┐
│ Redis (短期)     Milvus (长期)    Neo4j   │
│     ↓                ↓              ↓     │
│ AgentMemory     AgentMemory    独立运行   │
│                      ↓                    │
│               QA Memory (重复)            │
└───────────────────────────────────────────┘

问题: 存储层重复、无统一抽象
```

## 3. 改进目标

### 3.1 统一记忆架构

```
目标架构:
┌─────────────────────────────────────────────────────┐
│                  UnifiedMemoryManager               │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐   │
│  │ Working     │ │ Episodic    │ │ Semantic    │   │
│  │ Memory      │ │ Memory      │ │ Memory      │   │
│  │ (短期)      │ │ (情景)      │ │ (语义)      │   │
│  └─────────────┘ └─────────────┘ └─────────────┘   │
│         ↓               ↓               ↓          │
│  ┌─────────────────────────────────────────────┐   │
│  │           Knowledge Graph Layer             │   │
│  │         (实体、关系、推理)                   │   │
│  └─────────────────────────────────────────────┘   │
│         ↓               ↓               ↓          │
│  ┌─────────────────────────────────────────────┐   │
│  │          Unified Storage Layer              │   │
│  │    Redis + Milvus + (Optional Neo4j)        │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### 3.2 核心设计原则

1. **分层抽象**: 记忆类型 → 图谱层 → 存储层
2. **统一接口**: 所有记忆类型共享 MemoryBase 协议
3. **可插拔存储**: 支持 Redis/Milvus/Neo4j 任意组合
4. **智能索引**: 多路召回 (向量 + 关键词 + 图谱)
5. **生命周期管理**: 记忆创建、更新、压缩、遗忘
